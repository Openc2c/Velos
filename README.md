# Velos

Velos 是一个面向 Android 的弱网调试工具。它通过本地 VPN 方式接管流量，用来模拟限速、延迟、抖动、随机丢包和连续丢包等网络环境，方便在真机上复现“网络不好时会发生什么”。

这个仓库保存的是一套可以继续维护的实际工程，不是只拿来截图演示的半成品。现在这版已经把日常会用到的东西补得比较完整了：场景可以保存，配置可以导入导出，悬浮窗可以快速切换，启动时也能按目标应用去做调试。对开发、测试和联调来说，它已经是能直接上手的一套基础工具。

| 项目项 | 说明 |
| --- | --- |
| 平台 | Android |
| 开发语言 | Kotlin |
| UI | Jetpack Compose |
| 网络实现方式 | 基于 Android VPN Service 的本地流量控制 |
| 最低支持版本 | Android 8.0（API 26） |
| 当前版本号 | 2.0.0 |


## 现在有哪些功能

Velos 这套东西本质上就是为了省掉反复配环境的麻烦。你可以新建场景，也可以在已有场景上直接改参数继续用。常见的弱网项都已经做成了可视化配置，包括上下行带宽、延迟、延时抖动、随机丢包，以及更极端一些的连续丢包模式。

除了核心的网络参数外，这个版本还补上了几项更实用的日常能力。配置可以导出成备份文件，也可以重新导入；启动时可以按设置决定是否自动拉起目标应用；如果使用悬浮窗，已选配置会用颜色状态区分，但不会再额外加粗。关于页也已经改成独立页面，不再是简单的提示弹窗。

## 使用方式

首次使用时，应用会申请 VPN 和悬浮窗相关权限。授权完成后，可以先建立一个弱网配置，再选定要作用的目标应用，然后从主界面启动。如果你选择的是“全部应用”，应用会只启用弱网环境，不会额外拉起某个指定程序。

如果你习惯把常用方案分门别类地留存下来，可以把配置导出成 JSON 备份。换机、重装或者多人协作时，再通过导入功能恢复即可。对于需要反复切换不同网络场景的情况，悬浮窗入口会更顺手一些。

## 本地构建

项目就是标准的 Gradle Android 工程，直接用 Android Studio 打开仓库根目录就能继续做。默认情况下，即使本地没有签名信息，也照样可以正常编译和调试；如果环境里带上签名变量，`release` 构建就会自动走签名流程。

| 构建项 | 说明 |
| --- | --- |
| JDK | 17 |
| compileSdk / targetSdk | 35 |
| minSdk | 26 |
| 推荐构建命令 | `./gradlew assembleRelease` |


## 仓库结构

工程没有做复杂拆分，整体保持单应用结构，方便继续迭代。界面、配置和服务逻辑都在 `app` 模块内，工作流文件位于 `.github/workflows`。如果后面要继续扩展，例如增加更多弱网模型、增加日志面板或者补自动化测试，这个布局也足够继续演进。
